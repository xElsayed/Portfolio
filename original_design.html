Case Study: Wazuh SIEM

// Objective

To build and configure a complete SIEM solution using Wazuh to detect and automatically respond to a simulated attack by blocking the attacker's IP.

// Process

  1. Installed Wazuh server, indexer, and dashboard on an Ubuntu server.
  2. Deployed Wazuh agents on both Ubuntu (hosting Apache) and Windows endpoints.
  3. Configured "Active Response" to trigger a script blocking malicious IPs for 60 seconds.
  4. Simulated a web server attack from a malicious IP.

// Code Snippet (ossec.conf)

This is the most critical part. Below is the actual Active Response configuration used to block the attacker.


<!-- This is a placeholder. Paste your actual code here. -->
<ossec_config>
  <command>
    <name>firewall-block</name>
    <executable>firewall-cmd</executable>
    <expect>srcip</expect>
    <timeout_allowed>yes</timeout_allowed>
  </command>

  <active-response>
    <command>firewall-block</command>
    <location>local</location>
    <rules_id>100100</rules_id> <!-- Example Rule ID -->
    <timeout>60</timeout>
  </active-response>
</ossec_config>

// Outcome

Wazuh successfully detected the attack pattern, generated an alert, and automatically triggered the Active Response, blocking the attacker's IP from accessing the server. This project proved hands-on competence in SIEM deployment and automated incident response.
